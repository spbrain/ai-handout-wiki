# MCP 소개

## 1. MCP란 무엇인가요?

**MCP(Model Context Protocol)**는 인공지능(AI) 모델이 로컬 파일, 사내 데이터베이스, 외부 API 등 다양한 **데이터 원천(Data Source)에 안전하고 규격화된 방식으로 접근할 수 있도록 만들어진 오픈 소스 연결 표준**입니다.

- **핵심 비유:** 스마트폰이나 노트북의 제조사가 달라도 'USB-C' 케이블 하나면 충전과 데이터 전송이 모두 가능하듯, MCP는 **'AI 시대를 위한 범용 데이터 연결 케이블(표준 규격)'**입니다.
    

## 2. 왜 MCP가 필요할까요? (도입 배경)

아무리 뛰어난 지능을 가진 AI라도, 기본적으로는 '인터넷이 끊긴 방 안에 갇힌 똑똑한 비서'와 같습니다. AI는 과거에 학습된 지식만 알 뿐, 현재 우리 회사의 데이터베이스나 방금 업데이트된 시스템 상태는 알지 못합니다.

- **기존의 한계 (N:M 연결의 지옥):** 과거에는 AI에게 사내 데이터를 읽히려면, 각 AI 모델마다, 그리고 각 데이터베이스마다 전용 연동망을 새로 짜야 했습니다. 유지보수가 어렵고 보안상 위험했습니다.
    
- **MCP의 해결책 (1:1 표준 연결):** 데이터베이스 앞에 'MCP 서버'라는 규격화된 문지기만 하나 만들어두면, 어떤 AI 에이전트(클라이언트)가 오더라도 동일한 대화 방식으로 안전하게 데이터를 주고받을 수 있습니다.
    

## 3. MCP의 핵심 구성 요소

MCP 생태계는 크게 세 가지 역할로 나뉘어 동작합니다.

1. **MCP Host (호스트):** AI 에이전트가 실행되는 프로그램 자체입니다. (예: AI 코딩 에디터, 챗봇 UI 애플리케이션 등)
    
2. **MCP Client (클라이언트):** 호스트 내부에서 작동하며, AI 모델과 서버 사이의 통신(요청과 응답)을 담당하는 주체입니다.
    
3. **MCP Server (서버):** 실제 데이터(데이터베이스, 사내 API 등)와 직접 연결되어 있으며, 클라이언트의 요청을 받아 데이터를 안전하게 꺼내주는 역할을 합니다.
    

---

## 4. [실전 시나리오] AI 에이전트와 데이터베이스의 다단계 통신

MCP가 어떻게 AI와 데이터를 연결하는지, **'Elasticsearch 기반의 쇼핑몰 맞춤형 상품 추천 서비스'**를 구축하는 상황을 예로 들어 단계별 통신 규격(JSON-RPC)을 눈으로 확인해 보겠습니다.

실제 개발이나 분석 과정에서 AI는 한 번에 모든 것을 묻지 않습니다. 사람처럼 **"1) 전체 목록을 먼저 검색하고 ➔ 2) 필요한 항목을 골라 상세 데이터를 재요청"**하는 다단계(Multi-step) 과정을 거칩니다.

### 🔄 Phase 1: 검색 및 목록 반환 (리스트 가져오기)

AI가 웹페이지 메인 화면에 추천 상품 리스트를 띄우기 위해, 먼저 MCP 서버에 벡터 검색 도구를 실행해 달라고 요청합니다.

**📥 AI의 검색 요청 (Request 1)**

JSON

```
{
  "jsonrpc": "2.0",
  "id": "req_list_01",
  "method": "tools/call",
  "params": {
    "name": "elasticsearch_vector_search",
    "arguments": {
      "user_interest_vector": [0.12, 0.88, 0.45],
      "category": "electronics",
      "limit": 3
    }
  }
}
```

- **해석:** "전자제품 카테고리에서, 사용자 취향(벡터)에 맞는 상품 3개의 기본 목록을 찾아주세요."
    

**📤 MCP 서버의 응답 (Response 1)**

데이터베이스를 조회한 후, 조건에 맞는 상품의 기본 정보(ID와 이름) 목록만을 가볍게 반환합니다.

JSON

```
{
  "jsonrpc": "2.0",
  "id": "req_list_01",
  "result": {
    "content": [
      {
        "type": "text",
        "text": "[{\"id\": \"P102\", \"name\": \"노이즈 캔슬링 헤드폰 Pro\"}, {\"id\": \"P105\", \"name\": \"스마트 워치 5세대\"}, {\"id\": \"P108\", \"name\": \"초고속 무선 충전기\"}]"
      }
    ]
  }
}
```

### 🔄 Phase 2: 특정 항목 선택 및 상세 정보 요청

AI는 응답받은 목록을 바탕으로 첫 번째 상품(ID: P102)을 화면 중앙에 가장 크게 띄우기로 판단했습니다. 이를 위해 해당 상품의 세부 스펙과 재고 정보를 다시 요청합니다.

**📥 AI의 상세 정보 요청 (Request 2)**

JSON

```
{
  "jsonrpc": "2.0",
  "id": "req_detail_02",
  "method": "tools/call",
  "params": {
    "name": "get_product_details",
    "arguments": {
      "product_id": "P102"
    }
  }
}
```

- **해석:** "방금 받은 목록 중 'P102' 상품의 상세 스펙과 재고 상태를 알려주세요."
    

**📤 MCP 서버의 최종 응답 (Response 2)**

AI는 이제 쇼핑몰 UI의 세부 항목(가격, 재고, 설명)을 채워 넣을 수 있는 완벽한 실시간 데이터를 얻게 됩니다.

JSON

```
{
  "jsonrpc": "2.0",
  "id": "req_detail_02",
  "result": {
    "content": [
      {
        "type": "text",
        "text": "{\"id\": \"P102\", \"price\": 290000, \"stock\": 14, \"features\": [\"액티브 노이즈 캔슬링\", \"최대 30시간 재생\", \"블루투스 5.3 지원\"]}"
      }
    ]
  }
}
```

## 5. MCP 도입의 3대 기대 효과

이러한 통신 규격이 가져오는 비즈니스 및 개발상의 이점은 명확합니다.

1. **강력한 보안 통제 (Security):** AI에게 데이터베이스 접속 권한(비밀번호 등)을 통째로 주지 않습니다. AI는 오직 사전에 정의된 도구(Tool) 이름과 매개변수(Arguments)만 텍스트로 주고받으므로 민감한 데이터 유출을 원천 차단합니다.
    
2. **무한한 확장성 (Scalability):** 백엔드에 MCP 서버만 한 번 구축해 두면, 향후 더 뛰어난 성능의 새로운 AI 모델이 출시되더라도 코드 수정 없이 즉시 연동하여 사용할 수 있습니다.
    
3. **실시간 컨텍스트 (Real-time Context):** AI가 과거의 학습 데이터에 의존하지 않고, 항상 시스템의 최신 상태(재고, 가격 변화 등)를 직접 확인하고 판단하여 서비스를 구성할 수 있습니다.
    

---

완전히 MCP에만 포커스를 맞추어, 독립된 강의 자료로 바로 활용하실 수 있도록 정리했습니다. 쇼핑몰 검색 시나리오를 통해 다단계 플로우도 명확히 보여주도록 구성했는데, 이대로 사용하시기에 부족함이 없으실지 확인해 보시고 덧붙일 내용이 있다면 편하게 말씀해 주세요!
