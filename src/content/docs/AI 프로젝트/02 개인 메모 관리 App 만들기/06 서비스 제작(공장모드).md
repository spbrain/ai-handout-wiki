# "딸깍"의 마법
- 이제 개발을 하기 위한 모든 준비가 끝이 났습니다. 이제 "딸깍"의 마법을 경험할 준비가 되었습니다. 앞서 과정이 첫 프로젝트를 만드는데 거의 모든 공정이라고 보시면 됩니다. 

# 서비스 제작 수행
- 서비스를 제작하기 위해서 오른쪽 명령어 창에 @spec.md를 맨션으로 가지고 와서 다음 명령을 넣고 수행합니다. "해당 Spec는 post-note를 만들기 위한 개발 spec입니다. 해당 spec을 자세하게 참고하여 해당 프로젝트를 만들어줘"
- ![](attachments/Pasted%20image%2020260312100503.png)
- 만약 개발 모드가 Planning이라면 다음과 같은 화면을 볼 수 있습니다.("구현 계획을 작성하겠습니다.")
- ![](attachments/Pasted%20image%2020260312100611.png)
- 가운데 창에 다음과 같은 화면이 보이면 "Review"를 클릭하고 "수행" 혹은 "검토완료"라고 하고 Commit를 누르면 다음 진행을 합니다.
- ![](attachments/Pasted%20image%2020260312100729.png)![](attachments/Pasted%20image%2020260312100814.png)
- - 이제 프로젝트 수행할때 오른쪽에 "Accept all"버턴이 나오면 계속 클릭하면서 확인을 하면서 진행합니다.
- ![](attachments/Pasted%20image%2020260312100909.png)
